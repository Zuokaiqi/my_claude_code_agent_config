# 通用一键美化指南

## 触发场景

用户说"美化一下""排版好看点""加点样式""太丑了"等任何对现有文档排版不满意的情况。

## 美化检查清单

对任意飞书文档，按以下顺序检查并优化：

### 1. 标题层级

- [ ] 层级是否清晰？H1-H3 为主，不超过 H4
- [ ] 同级标题是否连续？避免出现 H1 → H3 跳级
- [ ] 标题是否有颜色区分？一级标题可用 `{color="blue"}`

### 2. 视觉节奏

- [ ] 大段文字是否超过 3 行？超过 → 拆分或加高亮块
- [ ] 相邻同类型块是否用 `---` 分隔？
- [ ] 是否有足够的留白？段落之间空一行

### 3. 高亮块（Callout）

关键信息必须用 Callout 突出：

```markdown
<callout emoji="💡" background-color="light-blue">
提示信息
</callout>
```

| 信息类型 | Emoji | 颜色 |
|---------|-------|------|
| 提示/说明 | 💡 | light-blue |
| 警告/注意 | ⚠️ | light-yellow |
| 错误/风险 | ❌ | light-red |
| 成功/达成 | ✅ | light-green |
| 重要信息 | 📌 | light-purple |

### 4. 分栏（Grid）

适合场景：
- 对比信息（左右/上下对比）
- 概览卡片（3-4 列并排）
- 步骤说明（2 列：步骤+说明）

```markdown
<grid cols="2">
<column>

**左侧标题**

左侧内容

</column>
<column>

**右侧标题**

右侧内容

</column>
</grid>
```

### 5. 表格优化

- 简单数据用标准 Markdown 表格
- 复杂单元格用 `<lark-table>`
- 表头加粗，数据行不加粗
- 数字右对齐或居中

### 6. 列表优化

- 步骤说明用有序列表
- 并列项用无序列表
- 嵌套不超过 2 层

### 7. 色彩运用

- 标题颜色不超过 2 种
- 强调文字用 `<text color="xxx">`，不滥用
- 同一段落内颜色不超过 2 种

### 8. 底部收尾

- 长文档底部加 `---` 分割线
- 可添加来源/时间戳/作者信息

## 一键美化流程

```
1. 读取原文档内容
2. 分析内容结构（标题、段落、列表、表格）
3. 识别内容类型（日报/会议/周报/其他）
4. 套用对应模板或通用美化规则
5. 添加视觉元素（Callout、Grid、颜色）
6. 检查排版细节（留白、对齐、层级）
7. 用 update-doc overwrite/replace_range 写入
```

## 禁忌

- ❌ 不要给每个字都加颜色
- ❌ 不要滥用 Callout（一页超过 5 个就太多了）
- ❌ 不要嵌套超过 2 层的 Grid
- ❌ 不要用超过 3 种不同颜色的标题
- ❌ 不要在表格里再放表格
