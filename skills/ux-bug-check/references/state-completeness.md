# 状态完整性类

## 视角

任何展示数据/承载交互的组件都不只有「正常」一种状态。空、加载、错误、边界这些状态做完整了吗。

## 五态枚举

| 状态 | 触发条件 | 必备元素 |
|------|---------|---------|
| 正常 | 有数据、操作成功 | 实际内容 |
| 空 | 用户从未创建、数据被清空、筛选无结果 | 解释 + 下一步引导（参考 ux-writing 空状态文案） |
| 加载 | 异步请求中 | 骨架屏或菊花 + 「加载中...」 |
| 错误 | 网络断、超时、4xx、5xx、权限不足 | 错误说明 + 重试入口 |
| 边界 | 长文本溢出、空数据塌陷、超长列表、数字超大 | 截断、省略、分页、缩写 |

## 常见遗漏

- 列表组件只做了正常态，空状态一片白
- 异步组件没loading，白屏后突然出现内容（CLS抖动）
- 错误态用 alert 弹窗（粗暴），不是组件内的友好提示
- 长文本撑破布局（标题、用户名、评论内容、商品名）
- 「下一页」按钮在没下一页时还能点
- 数字字段「1234567890」溢出容器
- 头像加载失败后是破图，不是 fallback
- 表单字段为空时显示 `undefined` 或 `null`

## 错误态细分

| 错误类型 | 文案方向 | 必备入口 |
|---------|---------|---------|
| 网络断 | 网络似乎断了 | 重试 / 下拉刷新 |
| 超时 | 服务响应慢 | 再试一次 |
| 4xx权限 | 这个操作需要某权限 | 登录 / 申请权限 |
| 4xx找不到 | 这个内容不存在或已删除 | 返回 / 回首页 |
| 5xx服务 | 服务暂时不可用 | 反馈 / 稍后再试 |
| 数据冲突 | 别人刚改过这条数据 | 刷新看最新版 |

## 静态分析能看到的

- 列表渲染前有没有 `length === 0` 或 `!data` 判断
- 异步组件有没有 loading 分支
- try/catch 后是否有错误UI渲染
- 文本字段有没有 ellipsis、truncate、line-clamp 类样式
- 头像/图片有没有 onError fallback

## 静态分析看不到的

- 空状态视觉是否友好
- 加载骨架是否拟合内容
- 错误态视觉是否过于粗暴

实跑时观察：清空数据看空状态、断网看错误态、灌超长内容看边界态

## 改进方向词

- 给组件加空状态视觉 + 文案 + 引导按钮
- 加 loading 骨架屏避免 CLS
- 错误态在组件内友好显示，不用粗暴alert
- 文本字段加 ellipsis 或多行截断
- 列表加空状态判断、加分页边界判断
- 图片头像加 fallback
- 数字字段考虑千分位和单位缩写（10K、1.2M）
