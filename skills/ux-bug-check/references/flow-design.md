# 流程类

## 视角

多步操作、跨页面流程、表单填写。用户走这一连串动作时顺不顺、被不被打断、能不能恢复。

## 检查项

1. 多步流程标了「第N步/共M步」吗
2. 中途打断（要求登录、网络断、误关）会保留已填信息吗
3. 完成核心动作后有下一步引导吗
4. 可以中途退出/暂停吗
5. 退出时数据保留吗
6. 步骤跳转可逆吗（上一步可回）
7. 表单字段顺序符合用户填写心理（不跳跃）

## 常见反例

- 5步表单没进度条，用户填到第3步以为快完了
- 表单填到一半要求登录，登录后表单清空
- 注册成功后没有任何引导，用户不知道下一步做什么
- 多步流程「下一步」按钮在底部但「上一步」不存在
- 弹窗一关就消失，没保留状态
- 表单字段顺序奇怪（手机号在最下面、姓名在最上面但要先填身份证号）
- 完成支付后直接关闭页面，没有「查看订单」入口
- 关键操作完成后没有可分享的瞬间

## 静态分析能看到的

- 多步流程组件里有没有 step indicator / progress bar
- 表单状态有没有持久化（localStorage、sessionStorage、后端 draft）
- 路由跳转后是否有数据传递（query、state、context）
- 步骤组件是否支持双向跳转

## 静态分析看不到的

- 真实使用时打断恢复的体验
- 步骤之间的视觉过渡

实跑时观察：填表到一半刷新页面，已填内容还在吗；走完一个流程，结束页是否引导下一步

## 改进方向词

- 多步流程加进度指示（步骤条或「第N/共M步」）
- 关键流程加表单状态保存（localStorage 或后端草稿）
- 完成后给下一步引导（去看XX、继续创建、分享）
- 步骤跳转双向（上一步保留已填、下一步可跳）
- 退出确认（已填内容不丢失）
- 字段顺序按用户填写直觉排（基础信息→详细信息→可选信息）
- 关键节点提供可分享入口（订单成功后能截图分享、文章发布后能转链接）
