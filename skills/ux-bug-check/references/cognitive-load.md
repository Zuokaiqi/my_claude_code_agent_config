# 认知负担类

## 视角

用户看这个界面要不要琢磨、要不要回想、要不要去对照别的地方才能用。

## 检查项

1. 主次操作视觉权重区分明显吗（主按钮 vs 次按钮）
2. 链接/按钮/纯文本能一眼分清吗
3. 信息层级清晰吗（重要的有没有被淹没）
4. 同语义不同词在不同位置出现吗（「订单」vs「购买记录」vs「交易历史」）
5. 用了用户不熟的术语没解释吗
6. 同一个功能在不同入口长得不一样吗
7. 表单字段标签是否清晰指代

## 常见反例

- 主操作和次操作都是相同样式的按钮（用户不知道点哪个）
- 链接像按钮、按钮像链接（用户不知道哪个能点）
- 表单里关键字段和次要字段视觉权重一样
- 同一个功能在不同入口叫不同名字
- 抛业内术语没有解释（参考 ux-writing 的术语翻译）
- icon-only 按钮没有 tooltip，用户要悬停才知道是什么
- 表单字段 label 模糊（「数据」「内容」「描述」）

## 静态分析能看到的

- 按钮组件的 variant 用法（primary/secondary/text）
- 链接 vs 按钮的标签使用（`<a>` vs `<button>`）
- 同义词全局搜索（Grep 项目里所有「订单」「购买记录」等候选词）
- icon-only 按钮有没有 aria-label / tooltip / title

## 静态分析看不到的

- 视觉权重的真实层次感
- 信息密度对认知的实际影响

实跑时观察：5秒内能不能找到主操作；同类操作在不同页面长得一样吗；表单字段一眼能懂吗

## 改进方向词

- 主操作填充色高对比，次操作描边或文字
- 链接用蓝色或下划线明确标识，按钮用块状视觉
- 关键字段用视觉锚点（粗体、icon、对齐边线）
- 全局术语统一（建立术语表，Grep 替换不一致的同义词）
- 术语首次出现时附小问号或解释
- icon-only 按钮加 tooltip 或 aria-label
- 表单 label 用具体名词（「公司名称」不是「名称」）
