# 用户控制感类

## 视角

用户感觉是自己在主导这个产品，还是被产品推着、骗着、按着走。

## 检查项

1. 用户能撤销吗（关键操作的撤销路径）
2. 用户能退出当前流程吗（弹窗、引导、多步表单的退出口）
3. 有没有强弹窗、强引导、强订阅（用户必须做某动作才能继续）
4. 默认勾选了用户没主动想要的东西吗（默认订阅、默认同意、默认升级）
5. 通知/弹窗用户能关掉吗、关掉后会再来骚扰吗
6. 用户能控制频率吗（推送、通知、邮件的频率设置）

## 常见反例

- 弹窗没有关闭按钮或关闭按钮藏角落
- 引导弹窗强制「下一步」走完才能用
- 注册时默认勾选「订阅营销邮件」「同意第三方共享」
- 升级提示无「以后再说」选项
- 通知关掉后明天又来一次
- 设置页找不到「关闭推送」入口
- 操作流程里突然插入广告或推荐，没有跳过按钮
- 自动播放视频/音频，没有立即关闭入口
- 默认开启地理位置/通讯录权限请求，不解释为什么需要

## 静态分析能看到的

- 弹窗组件是否有 close / dismiss / cancel 入口
- 默认值（checkbox checked、radio default、switch on）
- 引导流程是否有 skip / 跳过 按钮
- 设置页是否有对应的关闭开关

## 静态分析看不到的

- 弹窗关闭按钮的视觉可发现性
- 默认勾选的实际诱导性
- 频率控制的真实效果

实跑时观察：每个弹窗都能轻松关掉吗；默认选项是否倾向用户利益；引导流程能跳过吗

## 改进方向词

- 关键操作加撤销路径（删除、清空、退款、提交）
- 弹窗有明显的关闭按钮和「以后再说」入口
- 默认不勾选用户没主动选的项（除了协议条款这类必须勾选的）
- 通知关闭后有时间窗（一周不再提醒）或永久关闭选项
- 引导可跳过（明显的「跳过引导」按钮）
- 设置页提供完整的频率/类型控制
- 权限请求前解释为什么需要（不是上来就弹系统权限框）
- 自动播放默认关闭，用户主动触发
