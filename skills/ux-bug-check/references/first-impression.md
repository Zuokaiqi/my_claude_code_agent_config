# 第一印象类

## 视角

用户进入这个页面/组件的前3秒、前10秒、前30秒，能不能看懂这是什么、能找到想要的、会留下来。

## 检查项

1. 进入页面3秒内能看懂这是什么吗
2. 主操作有没有视觉锚点（用户一眼能找到主按钮在哪）
3. 信息密度会不会让人扫一眼就想划走
4. 标题/卡片标题清晰描述当前是什么页面/什么内容吗
5. 首屏默认状态是不是对齐了最高频用户需求

## 常见反例

- 首屏被弹窗/引导/广告挡住，用户先要躲一次才能用
- 首屏没有标题，用户不知道自己在哪个页面
- 主操作和次操作视觉权重一样，用户不知道点哪个
- 信息堆满，无视觉呼吸（密集不等于高效）
- 异步加载没有骨架屏，白屏后突然出现内容
- 进入页面默认状态不是用户最常用的（如默认显示「访问量」但用户每天看「销售额」）
- 进入即弹窗强引导（新手教程、新功能介绍）

## 静态分析能看到的

- HTML/JSX 顶层结构里有没有 modal/overlay 在首屏强制出现
- 主操作按钮是否有 primary 类或视觉权重区分类名
- 首屏区域内的元素数量（粗略判断密度）
- loading 处理是否用骨架屏（skeleton/Skeleton/placeholder）

## 静态分析看不到的

- 真实首屏视觉冲击力
- 文字密度的真实观感
- 信息层级的视觉效果

实跑时观察：截图首屏 → 把图缩小到30% → 还能看出主操作在哪、当前在做什么吗

## 改进方向词

- 增加首屏视觉锚点（主标题+主按钮视觉权重提高）
- 默认状态对齐最高频用户需求
- 异步内容用骨架屏，不要白屏
- 弹窗对登录态用户不弹或低频弹
- 主次操作视觉权重明确分级
