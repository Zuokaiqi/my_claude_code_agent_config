# 操作反馈类

## 视角

用户点了一下、滑了一下、按了一下，能不能立刻感受到「我刚才那个动作起作用了」。

## 检查项

1. 按钮按下去有视觉反馈吗（active态、按下态、波纹）
2. 异步操作有loading吗（按钮变「保存中...」或外层loading）
3. 操作完成有明确「成功」反馈吗（toast / 按钮变「已保存」/ 页面跳转）
4. 错误时反馈出现在哪、看得见吗
5. 长操作（>3秒）有进度提示吗
6. 不可逆操作有二次确认吗（这块也归第7类情绪，重叠属于正常）

## 常见反例

- 按钮按下去毫无变化，用户重复点击
- 异步保存按钮没loading，用户不确定点没点上
- 操作成功后无任何提示，用户不确定有没有成功
- 错误toast 1秒消失，用户没看清错误内容
- 上传/下载长操作只显示菊花图标，不显示进度
- 表单提交后跳转太快，用户没看到「保存成功」就到下一页
- 删除/清空无视觉确认（动画或文字），用户怀疑有没有真删

## 静态分析能看到的

- 按钮组件有没有 `:active` / pressed 样式或类名
- 异步处理代码里有没有 setState loading / setLoading / pending 之类的状态变更
- 提交后是否有 toast / message / notification 调用
- 错误处理是否有 catch 后的用户反馈代码

## 静态分析看不到的

- 反馈动效的真实流畅度
- toast 持续时间是否合适
- 按钮按下态的视觉清晰度

实跑时观察：按主要按钮，0.1秒内有没有视觉变化；异步操作期间按钮状态如何；成功后有没有持续2秒以上的可见反馈

## 改进方向词

- 按钮加按下态（active 视觉变化）
- 异步操作加loading（按钮内或外层）
- 成功反馈用 toast 或就近文案（按钮变「已保存」）
- 错误信息持续显示直到用户操作（不自动消失）
- 长操作显示进度（百分比、步骤、文件名）
- 不可逆操作加二次确认 modal
