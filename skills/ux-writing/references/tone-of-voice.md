# 语气一致性规则

## 核心原则

1. 「你」和「您」全产品统一，不混用
2. 专业度统一：面向C端的不用业内黑话，面向B端可保留术语
3. 称呼方式一致：「亲」「同学」「用户」全局统一或不用
4. 标点和符号一致：句末用不用句号、感叹号节制

## 「你」vs「您」选择

| 选「您」 | 选「你」 |
|---------|---------|
| 偏正式、距离感、商务 | 亲切、年轻、消费 |
| 银行、政务、企业SaaS | 社交、内容、电商、工具 |
| 用户年龄偏大、严肃场景 | 用户年龄轻、休闲场景 |

选定后全局不混用：

- 反例：同一界面里「您好」和「你的订单」并存
- 正例：通篇「你」或通篇「您」

自检方法：Grep 项目里所有「您」和「你」出现位置，对照看是否一致。

## 业内黑话翻译

C端用户不一定懂业内术语，需要翻译：

| 黑话 | 翻译 |
|------|------|
| 实名认证 | 用身份证验证一下身份 |
| 该订单已核销 | 这张券已经用过 |
| 冲账成功 | 退款已到账 |
| 已脱敏 | 关键信息已隐藏（保护隐私）|
| 风控拦截 | 安全检查没通过，看看是不是这些原因 |
| KYC验证 | 身份验证 |
| LTV / GMV | （不出现在C端界面） |
| 漏斗 / 转化率 | （不出现在C端界面） |
| SLA | 服务保障 |

B端SaaS、专业工具可保留术语，但首次出现时附小问号或解释。

## 标点节制

- 句末感叹号不连发：「保存成功！太棒了！」反例，改「已保存」
- 省略号不滥用：「正在加载……请耐心等待……」反例，改「加载中」
- 中文语境用中文标点：，。：；？！，不混入英文标点
- 句末是否带句号统一一种风格（短句通常不带）

## 表情符号

| 产品调性 | 使用建议 |
|---------|---------|
| 工具型（dashboard、企业SaaS、开发工具） | 几乎不用 |
| 社交型、内容型 | 按调性使用 |
| 儿童、教育 | 可适度使用 |
| 金融、医疗 | 不用 |

避免：错误信息里加哭脸、成功状态加 🎉🎉🎉、加载中加 ⏰。

## 一致性自检

写新文案前 Grep 项目里同类位置的现有文案：

| 检查项 | Grep 关键词 |
|--------|------------|
| 你/您混用 | 「您」「你」（数出现频次） |
| 同操作的反馈语气 | 「成功」「失败」「保存」「删除」 |
| 异常处理语气 | 「错误」「失败」「异常」「不可用」 |
| 称呼一致 | 「亲」「同学」「用户」「主人」 |

发现不一致时，跟项目里多数的那种保持一致，不要单独立一个新风格。

## 不要做的事

- 不要装高级：「为您精心准备」「品质臻享」是反例
- 不要装亲切：「亲，您看下哦」「主人」是反例（除非品牌调性确实如此）
- 不要装专业：抛一堆英文缩写
- 不要装可爱：成年人产品里随便加颜文字 (｡◕‿◕｡)
- 不要在严肃场景幽默（金融、医疗、法律的错误信息别玩梗）
