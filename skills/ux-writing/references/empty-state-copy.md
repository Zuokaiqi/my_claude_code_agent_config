# 空状态文案规则

## 核心原则

1. 不止于「暂无数据」「列表为空」「无内容」
2. 解释为什么是空的：是新用户、是筛选条件、是已删除
3. 给下一步建议：「点击右上角创建第一篇文章」
4. 区分「空」和「错」：空是正常状态，错是异常状态，文案语气要区分

## 反例对照

| 反例 | 改进 |
|------|------|
| 暂无数据 | 还没有订单，去逛逛吧 / 你的订单会显示在这里 |
| 列表为空 | 还没创建过任务，点右上角「+」开始 |
| 无内容 | 还没人发表评论，第一个聊聊？ |
| 搜索无结果 | 没找到「xxx」相关结果，换个关键词试试 |
| 加载失败 | 网络似乎断了，下拉刷新（这是错态不是空态，语气要区分） |
| 暂未开放 | 这个功能正在路上，{预计上线时间} |
| 数据加载中 | （这是loading态，不该出现在空状态文案位置） |

## 不同场景的空状态

### 新用户首次进入

欢迎+功能价值+开始按钮：

```
欢迎使用 XX 笔记
在这里记录想法、整理灵感
[新建第一篇笔记]
```

### 筛选/搜索无结果

告诉「没找到的是什么」+ 提示换关键词：

```
没找到「{关键词}」相关结果
试试更宽泛的词，或调整筛选条件
[清除筛选]
```

### 已清空/全部完成

带成就感的反馈：

```
你已处理完所有待办
喝杯水休息一下
```

### 历史性空（订单历史、消息记录等）

预告会有内容：

```
还没有订单
你的订单会显示在这里
[去逛逛]
```

### 权限性空（看不见但实际存在）

说明原因 + 获取路径：

```
这个分组里的内容你还没有权限查看
找管理员申请权限
```

## 文案语气

| 用户类型 | 语气 | 例 |
|---------|------|----|
| 新用户 | 欢迎、鼓励 | 欢迎，从创建第一篇开始 |
| 老用户 | 克制、不啰嗦 | 还没有新消息 |
| 异常空（应该有数据却没有） | 道歉+排查路径 | 数据加载出了问题，刷新或反馈 |
| 任务完成型空 | 成就感、舒缓 | 全部搞定，今天辛苦了 |

## 空状态的三件套

完整空状态包含：

```
[插图或图标]           ← 视觉锚点，避免一片白
[一句话状态说明]        ← 这里为什么是空的
[一句话引导]           ← 给用户下一步做什么
[行动按钮]            ← 直接可点
```

只有文字没有按钮的空状态，用户读完不知道下一步做什么，会产生「这页废了」的印象。

## 不要做的事

- 不要用四字成语堆砌（「空空如也」「一无所有」）
- 不要营销腔（「赶紧创建吧」「快来体验」）
- 不要责怪用户（「您还没有添加任何内容」）
- 不要把空状态做成广告位（在空状态里推其他功能）
