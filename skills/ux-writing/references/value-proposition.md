# 价值主张写法（H1 + 副标题）

H1 是落地页最重要的一行字。命中具体人物的具体痛点 > 取悦所有人。读者 2 秒内做出去留判断，靠的就是 H1。

## H1 的硬性要求

1. **2 秒能读完**：超过 30 字就过长
2. **写给具体的人**：不是"用户"、"团队"、"开发者"，而是"周末做独立产品的开发者"、"被催着写文案的工程师"
3. **写 outcome 不写 feature**：用户雇你这个产品是为了完成某个 job（JTBD），H1 应该命中那个 job，不是描述功能模块
4. **抄到竞品页面上不成立**：如果你的 H1 改个产品名贴到竞品页面也通顺，那就是太泛
5. **不用形容词凑数**：避免「高效」「智能」「极致」「卓越」「专业」

## 5 个价值主张公式

### 公式 1：Like X for Y（类比已知）

借用读者已经熟悉的概念，告诉他们这是给谁的版本。

- 「Like Evernote for pro writers」（Airstory）
- 「Like Slack for sales teams」
- 「Like Figma for video」

适合：新品类、需要快速建立认知的早期产品

### 公式 2：The [产品类] that [独特做法/outcome]

承认自己属于某个品类，但讲清楚做法上的差异化。

- 「The issue tracker you'll enjoy using」（Linear）—— issue tracker 是已知品类，enjoy 是差异化
- 「The CRM your sales team won't fight you on」
- 「The notes app that doesn't sync your data to anyone's server」

适合：成熟品类里的差异化产品

### 公式 3：For [audience] who [当下的痛点]，[产品] is [品类] that [benefit]

完整价值主张句式，长但信息密度高。可用于 H1 + 副标题组合。

- 「For Notion power users who hate slow loading，FastDoc is a docs app that opens instantly even with 5000 pages」
- 「For PM who write 5 PRDs a week，PRDFlow is a writer that turns bullet points into shipped specs」

适合：副标题或 hero 区下方的展开

### 公式 4：My product is the one that ___

填空法。强迫自己说出唯一性。如果填出来的句子也适合竞品，重写。

- ToneShift 是唯一一个只改语气不改结构的 AI 写作工具
- Linear 是唯一一个让工程师主动打开 issue tracker 的产品

适合：内部对齐用，也可以做副标题

### 公式 5：JTBD 式 ── Help [audience] [verb] [outcome] when [circumstance]

强调"在什么情境下"用户雇这个产品。

- 「Help indie hackers ship landing page copy when they only have weekend hours」
- 「Help engineers explain technical changes when the CEO needs an email in 30 minutes」

适合：场景驱动的产品

## 案例拆解

| 产品 | H1 | 拆解 |
|------|-----|------|
| Linear | The issue tracker you'll enjoy using | 承认是 issue tracker（不装作新品类）+「enjoy」一个简单词对比所有臃肿工具 |
| Stripe | Payments infrastructure for the internet | 用「infrastructure」这个开发者懂的词 + 不说支付公司、说支付基础设施 |
| Vercel | The Frontend Cloud | 4 个字造一个新词，框定整个品类 |
| Notion | Write, plan, share. With AI at your side. | 三个动词排比 + 「at your side」而非「powered by」体现陪伴感 |
| ToneShift（示例） | 把你写的草稿，改成符合品牌调性的成品 | 写给「自己写过草稿，不满意成品」的人 |

## 副标题作用

H1 命中具体人 + 一个 outcome；副标题做四件事：

1. **细化谁**：把 H1 里的"开发者"具体到"周末做独立产品的开发者"
2. **细化做什么**：H1 说"改文案"，副标题说"改语气保留结构"
3. **细化怎么做**：差异化路径（"用你过往的写作样本学语气"）
4. **细化跟谁不一样**：明确划出跟竞品的边界（"普通 AI 写作工具给你新文案，我们只改你已有的"）

副标题长度 60-120 字，2-4 句。不要重复 H1 的字。

## H1 黑名单

### 中文 AI 腔禁用词

| 禁用 | 为什么 |
|------|--------|
| 一站式 | 万能词，没人会说自己是「分阶段式」 |
| 智能化 | 所有 AI 产品都说自己智能 |
| 赋能 | 翻译自 empower，AI 重灾区 |
| 打造 | 滥用形容词，「打造极致体验」类 |
| 解锁 | unlock 翻译腔 |
| 沉浸式 | 没人能解释什么是沉浸 |
| 重新定义 | 你重新定义不了什么 |
| 不仅是X，更是Y | 营销稿套话 |
| 为...而生 | born for，翻译腔 |
| XX新生态 | 凑词 |
| 全方位 | 等于没说 |
| 一键 | 99% 的「一键」都需要不止一键 |

### 翻译腔结构

- 「让我们」「让我们一起」（let us）
- 「为您打造」「为您而设计」（for you）
- 「您将拥有」（you will have）
- 「我们将」（we will）
- 「您的不二选择」（your go-to）

### H1 反例 vs 正例

| 反例 | 正例 |
|------|------|
| 智能化AI写作助手，赋能内容创作 | 把你写的草稿，改成符合品牌调性的成品 |
| 一站式企业知识管理解决方案 | 给中小团队的 wiki，工程师建好后 PM 也能维护 |
| 重新定义团队协作 | The issue tracker your team won't avoid |
| 打造沉浸式学习体验 | 跟着做完 6 个 agent 项目，结课时手里有可上线的产品 |

## 自检清单

写完 H1 后过一遍：

- [ ] 抄到任一竞品落地页上还成立吗？成立就太泛
- [ ] 30 字以内读完了吗？
- [ ] 有具体的人吗？还是「用户」「团队」「企业」？
- [ ] 有具体的 outcome 吗？还是「提升」「优化」「赋能」？
- [ ] 黑名单词全部不在 H1 里吗？
- [ ] 朗读一遍是不是人会说的话？「让我们一起开启智能化新生态」是 AI 写的
- [ ] 删掉之后用户还能从副标题/截图猜出产品是什么吗？应该不能（说明 H1 信息密度足够）

## 副标题自检

- [ ] 没有重复 H1 的字
- [ ] 至少出现一个具体的人/事/物
- [ ] 至少出现一个跟竞品的差异点
- [ ] 字数在 60-120 字之间
- [ ] 不出现「为您」「您将」「让我们」
